# ClaudeBar v1.0 Release — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship ClaudeBar as a public GitHub repo with a GitHub Actions workflow that builds, zips, and publishes a release on every version tag.

**Architecture:** `release.sh` bumps version + tags locally; pushing the tag triggers a `macos-latest` CI job that builds `ClaudeBar.app`, zips it, and creates a GitHub release with the zip attached and CHANGELOG notes as the body.

**Tech Stack:** Swift/AppKit, bash, GitHub Actions (`softprops/action-gh-release@v2`), `gh` CLI for repo creation.

### Task 1: Create `release.sh`

**Files:**
- Create: `release.sh`

- [ ] **Step 1: Create the script**

```bash
cat > release.sh << 'EOF'
#!/bin/bash
set -e

VERSION="$1"
if [ -z "$VERSION" ]; then
    echo "Usage: ./release.sh <version>  (e.g. ./release.sh 1.0.0)"
    exit 1
fi

# Abort if working tree is dirty
if ! git diff --quiet || ! git diff --cached --quiet; then
    echo "Error: working tree has uncommitted changes. Commit or stash first."
    exit 1
fi

# Verify CHANGELOG has a section for this version
if ! grep -q "^## \[$VERSION\]" CHANGELOG.md; then
    echo "Error: no '## [$VERSION]' section found in CHANGELOG.md"
    echo "Add release notes to CHANGELOG.md first, then re-run."
    exit 1
fi

# Update CFBundleVersion and CFBundleShortVersionString in build.sh
sed -i '' "/CFBundleVersion</{n; s|<string>[^<]*</string>|<string>${VERSION}</string>|;}" build.sh
sed -i '' "/CFBundleShortVersionString/{n; s|<string>[^<]*</string>|<string>${VERSION}</string>|;}" build.sh

echo "Updated build.sh to version $VERSION"

# Commit, tag, push
git add build.sh
git commit -m "chore: release v${VERSION}"
git tag "v${VERSION}"
git push origin main
git push origin "v${VERSION}"

echo ""
echo "✓ Tagged v${VERSION} and pushed — CI will build and publish the release."
EOF
chmod +x release.sh
```

### Task 2: Create `.github/workflows/release.yml`

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: Create the workflows directory and file**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: Write the workflow**

Create `.github/workflows/release.yml` with this exact content:

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: macos-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Generate icon
        run: swift make_icon.swift

      - name: Build app
        run: bash build.sh

      - name: Re-sign ad-hoc
        run: |
          xattr -cr ClaudeBar.app
          codesign --force --deep --sign - ClaudeBar.app

      - name: Zip app
        run: zip -r "ClaudeBar-${{ github.ref_name }}.zip" ClaudeBar.app

      - name: Extract release notes
        id: notes
        run: |
          VERSION="${{ github.ref_name }}"
          BARE="${VERSION#v}"
          NOTES=$(awk "/^## \[${BARE}\]/{found=1; next} found && /^## \[/{exit} found{print}" CHANGELOG.md)
          {
            echo "body<<EOF"
            echo "$NOTES"
            echo "EOF"
          } >> "$GITHUB_OUTPUT"

      - name: Create GitHub release
        uses: softprops/action-gh-release@v2
        with:
          name: "ClaudeBar ${{ github.ref_name }}"
          body: ${{ steps.notes.outputs.body }}
          files: "ClaudeBar-${{ github.ref_name }}.zip"
```

- [ ] **Step 1: Replace README with updated content including Install section**

Overwrite `README.md` with:

```markdown
# ClaudeBar

macOS menubar app that shows your Claude Code token usage at a glance.

## What it does

Displays two lines in the macOS menubar:
- **Top:** session usage % and time until session reset
- **Bottom:** weekly usage % and time until weekly reset

Reads your token from the macOS Keychain (same place Claude Code stores it) and polls the Anthropic usage API every 60 seconds.

## Install

### Download (recommended)

1. Download `ClaudeBar-vX.Y.Z.zip` from [Releases](../../releases/latest)
2. Unzip and drag `ClaudeBar.app` to `/Applications/`
3. First launch — Gatekeeper will block it (ad-hoc signed, not notarized):
   ```bash
   xattr -cr /Applications/ClaudeBar.app
   open /Applications/ClaudeBar.app
   ```
   Or: right-click → Open → Open anyway.

Use the menubar icon menu to enable **Launch at Login**.

### Build from source

#### Prerequisites

- macOS 13+
- Xcode command line tools: `xcode-select --install`
- Claude Code installed and authenticated (token stored in Keychain)

#### Build

```bash
# Regenerate icon (only needed when AppNameIcon.webp changes)
swift make_icon.swift

# Build ClaudeBar.app
bash build.sh

# Install
cp -r ClaudeBar.app /Applications/
xattr -cr /Applications/ClaudeBar.app
open /Applications/ClaudeBar.app
```

## Releasing a new version

```bash
# 1. Add release notes under a new ## [X.Y.Z] section in CHANGELOG.md
# 2. Run:
./release.sh X.Y.Z
```

This bumps the version in `build.sh`, commits, tags, and pushes. GitHub Actions builds and publishes the release automatically.
```
